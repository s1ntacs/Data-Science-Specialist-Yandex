# NorthCart Batch Features Pipeline

## Описание проекта

DAG для расчёта батч-признаков пользователей NorthCart. Пайплайн загружает данные из PostgreSQL, рассчитывает признаки для каждой пары (customer_id, run_date) с использованием окон 7 и 30 дней, и сохраняет результат в S3.

Ручной запуск через UI

Нажмите кнопку Trigger DAG (иконка Play)
Дождитесь успешного завершения задач
Запуск через CLI
bash
Расчёт признаков
DAG рассчитывает следующие батч-признаки для каждого пользователя:
Признаки событий (окна 7 и 30 дней):
Количество page_view событий
Количество add_to_cart событий
Конверсия page_view → add_to_cart
Конверсия add_to_cart → purchase
Количество уникальных товаров
Признаки сессий:
Количество сессий (7 и 30 дней)
Средняя длина сессии (30 дней)
Признаки заказов:
Количество заказов (30 дней)
Сумма заказов (30 дней)
Средний чек (30 дней)
Дней с последней покупки
Результат выполнения
После успешного выполнения DAG в S3 создаётся файл:
Где {дата} — значение переменной batch_features_run_date.
Проверка результата
Убедитесь, что DAG выполнился без ошибок
Проверьте наличие файла в S3 по пути run_date={дата}/batch_features.csv
Убедитесь, что файл не пустой и содержит все необходимые признаки

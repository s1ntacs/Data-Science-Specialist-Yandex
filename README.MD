# 1.Витрина данных и ad-hoc аналитика маркетплейса «ВсёТут»

## О проекте
Разработка витрины данных для маркетплейса и решение четырёх бизнес-задач на её основе с помощью SQL.

## Что сделано
- Спроектирована витрина данных `product_user_features` с агрегацией заказов, оплат и оценок
- Решены 4 ad-hoc задачи:
  - Сегментация пользователей по количеству заказов
  - Анализ топ-3 регионов
  - Помесячная когортная аналитика за 2023 год
  - Расчёт LTV и ключевых поведенческих метрик

## Стек
`PostgreSQL` · `SQL` · `CTE` · `Window Functions` · `GROUP BY` · `JOIN`

## Ключевые выводы
- 60 468 пользователей совершили только 1 заказ → проблема удержания
- Сегменты 6–10 и 11+ заказов крайне малы (всего 6 человек) — потенциал роста
- Средний чек повторных покупателей ниже, но частота покупок выше

# 2.Анализ лояльности пользователей Яндекс Афиши

## О проекте
Исследовательский анализ поведения пользователей билетной платформы с целью выявить факторы возврата и сформулировать рекомендации для отдела маркетинга.

## Что сделано
- Подключение к PostgreSQL и выгрузка данных через SQLAlchemy
- Предобработка: конвертация KZT → RUB, фильтрация выбросов, расчёт интервалов между заказами
- Построение профиля пользователя (20+ агрегированных признаков)
- EDA: распределения, сегментация по первому заказу, корреляционный анализ (`phi_k`)
- Визуализация через `matplotlib` и `seaborn`

## Стек
`Python` · `pandas` · `SQLAlchemy` · `PostgreSQL` · `matplotlib` · `seaborn` · `phik`

## Ключевые выводы
- 83% пользователей — одноразовые, точка входа — концерты (44%) и мобильные устройства (83%)
- Пользователи с 5+ заказами возвращаются в среднем на 10-й день
- Рекомендации: автоматизировать удержание на 10-й день, перераспределить бюджет с концертов на театр и стендап

# 3.Предсказание возрастной категории пользователя

## О проекте
Построение модели многоклассовой классификации, которая по анонимизированным поведенческим данным определяет возрастную категорию пользователя.

## Что сделано
- Объединение 6 таблиц (visits, users, ads_activity, surf_depth, primary_device, cloud_usage)
- Feature engineering: shares категорий сайтов, статистики по устройствам, облачному использованию
- Отбор признаков: `SelectKBest`, `RFE` (сокращение с 45 до 30 признаков)
- Обучение 3 моделей: LogisticRegression, SVC (linear), SVC (RBF)
- Подбор гиперпараметров через `GridSearchCV` + кросс-валидацию
- Упаковка модели в `Pipeline` + `ColumnTransformer` для продакшена
- Сохранение артефактов через `joblib`

## Стек
`Python` · `scikit-learn` · `pandas` · `numpy` · `joblib` · `Pipeline`

## Результат
- Лучшая модель: **SVC с RBF-ядром**
- F1-macro на тесте: **0.89**
- Модель готова к внедрению в рекламную сеть

# 4.Прогноз отмены бронирования в сети отелей

## О проекте
Разработка ML-модели для прогнозирования отмен бронирований с целью минимизации финансовых потерь и повышения загрузки отелей.

## Что сделано
- Выгрузка данных из PostgreSQL (`hotel_bookings`, `hotel_reviews`)
- Feature engineering: сезонность, длительность проживания, лояльность клиента, векторизация отзывов (`TF-IDF`)
- Обучение: **CatBoost** и **RandomForest**
- Оптимизация гиперпараметров через **Optuna**
- Временная кросс-валидация через `TimeSeriesSplit`
- Калибровка вероятностей через `CalibratedClassifierCV` (Platt Scaling)
- Подбор оптимального порога классификации по бизнес-метрике **Incremental Revenue (IR)**
- Анализ важности признаков

## Стек
`Python` · `CatBoost` · `scikit-learn` · `Optuna` · `PostgreSQL` · `SQLAlchemy` · `TF-IDF`

## Бизнес-результат
- Доля отмен: **23% → 4.8%** (снижение на 79%)
- Incremental Revenue: **227M → 342M RUB** (+51%)
- Загрузка отелей снизилась в пределах допустимого лимита (−4%)
- Топ-факторы отмен: низкая оценка проживания (50% важности) и короткие отзывы

# # Прогнозирование продаж для сети дарксторов «Прилавок»

## О проекте
Разработка ML-системы для прогнозирования недельных продаж по каждому отделу даркстора с целью оптимизации закупок, снижения дефицита и минимизации списаний.

## Что сделано
- Загрузка и объединение данных из PostgreSQL (`sales`, `stores`, `features`, `plan`)
- Feature engineering: лаговые признаки (`sales_1week_ago`, `sales_2week_ago`), скользящие средние, временные признаки
- Мониторинг стабильности признаков через PSI (Population Stability Index)
- Обучение модели **CatBoostRegressor** с early stopping
- Реализация production-пайплайна batch-инференса в **Apache Airflow**
- Сохранение/загрузка модели из **Yandex Cloud S3**
- Пост-обработка: клиппинг отрицательных прогнозов, батчевая запись результатов в БД

## Стек
`Python` · `CatBoost` · `pandas` · `scikit-learn` · `Apache Airflow` · `PostgreSQL` · `boto3` · `psycopg2`

## Бизнес-результат
- Качество модели: **R² = 0.997**, MAE = 343 ₽ на валидации
- Прогнозы для ~115K строк (плановые периоды) генерируются автоматически каждую неделю
- Пайплайн обрабатывает ~536K строк истории + плана за ~2 минуты
- Топ-факторы прогноза: продажи прошлой недели (46.6%), скользящие средние (24.3%), лаги 2-4 недели (15.1%)
